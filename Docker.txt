1. Introduction to Docker
1.1 What is Docker

Docker is a containerization platform

It allows applications to run in isolated environments (containers)

Ensures “works on my machine” problem is solved

1.2 Why Docker is Used

Consistency across environments

Lightweight compared to virtual machines

Faster deployment

Easy scalability

2. Docker Architecture
2.1 Components of Docker

Docker Client

Interface to interact with Docker

Commands like docker run, docker build

Docker Daemon

Runs in background

Manages containers, images

Docker Images

Read-only templates

Used to create containers

Docker Containers

Running instance of an image

Docker Registry

Stores images

Example: Docker Hub

3. Docker vs Virtual Machine
3.1 Key Differences
Feature	Docker	Virtual Machine
Size	Lightweight	Heavy
Boot time	Seconds	Minutes
OS	Shares host OS	Separate OS
Performance	Faster	Slower

7. Dockerfile
7.1 What is Dockerfile

A text file with instructions to build an image

7.2 Example
FROM ubuntu
RUN apt update

7.3 Build Image

8. Docker Volumes
8.1 What are Volumes

Used to persist data outside containers

8.2 Example

9. Docker Networking
9.1 Types of Networks

Host

10. Docker Compose
10.1 What is Docker Compose

Tool to manage multi-container applications

10.2 Example
